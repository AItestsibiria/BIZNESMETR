## §1. SUNO V4/4.5 API: ВСЁ, ЧТО МОЖНО ОТПРАВИТЬ  
  
### §1.1 Структурные теги в lyrics  
  
Suno распознаёт квадратные теги внутри текста и адаптирует ритмику/темп.  
  
**Стандартные секции:**  
```  
[Intro]  
[Verse 1] ... [Verse 2] ... [Verse 3]  
[Pre-Chorus]  
[Chorus]  
[Post-Chorus]  
[Bridge]  
[Instrumental Break] / [Solo]  
[Outro]  
[Fade Out]  
```  
  
**Динамические теги:**  
```  
[Soft] [Loud] [Whisper] [Powerful]  
[Slow build-up] [Drop] [Climax]  
[Drum fill] [Bass slap] [Synth lead]  
[Choir harmonies] [Backing vocals]  
```  
  
**Эмоциональные теги:**  
```  
[Mood: melancholic] [Mood: euphoric] [Mood: nostalgic] [Mood: tense]  
[Tempo: 90 bpm] [Tempo: 140 bpm]  
[Key: A minor] [Key: C major]  
```  
  
**Пример lyrics с тегами для свадебного трека:**  
```  
[Tempo: 75 bpm] [Mood: romantic] [Key: D major]  
  
[Intro: soft piano]  
  
[Verse 1]  
Тёплый свет в окошке нашего дома...  
Я искал тебя в каждом дне.  
  
[Pre-Chorus: building strings]  
И вот этот день настал, любимая моя...  
  
[Chorus: powerful, choir harmonies]  
Я выбираю тебя — на всю жизнь.  
В горе, в радости, в сердце моём...  
  
[Verse 2]  
...  
  
[Bridge: soft, intimate]  
Помнишь, как мы встретились впервые...  
  
[Chorus: triumphant, full band]  
...  
  
[Outro: fade with piano]  
```  
  
### §1.2 Style weighting (несколько жанров одновременно)  
  
```  
style: synthwave 0.7, lofi hip-hop 0.3, female vocals  
```  
  
или развёрнутый JSON в нашем API:  
```json  
{  
  "styles": [  
    { "name": "synthwave", "weight": 0.7 },  
    { "name": "lofi hip-hop", "weight": 0.3 }  
  ],  
  "vocals": { "gender": "female", "weight": 0.8 }  
}  
```  
  
**Бизнес-применение:** виральные гибриды («синтвейв-частушка», «трэп-блатняк»), которые невозможно ни в одной радиостанции — отличный контент для соцсетей.  
  
### §1.3 Negative prompts  
  
`negativePrompt: "harsh, screaming, electronic distortion"` — Suno избегает этих характеристик.  
  
**Бизнес-применение:** для свадебных и подарочных треков убираем грубость, для корпоративов — грусть.  
  
### §1.4 Persona ID (стабильный голос между треками)  
  
После генерации трека Suno возвращает `task_id` и опционально `persona_id` (вокальный отпечаток). При следующей генерации можно передать `persona_id`, и голос будет тот же.  
  
**Бизнес-применение:**  
- **Авторская подписка.** Юзер создаёт «своего исполнителя», все его треки — этим голосом. **+400 ₽** за привязку persona к аккаунту.  
- **Серия песен в подарок.** Свадебный плейлист «Наш голос» — 5 треков одним голосом. **+1500 ₽** за серию против 5 разовых.  
  
### §1.5 Extend (продление трека)  
  
`POST /generate/extend { source_audio_id, extend_at_sec, prompt? }` — продлевает существующий трек в музыкальном развитии.  
  
**Бизнес-применение:** трек получился 1:50, юзер хочет 3:30. **+149 ₽** за extend без новой генерации с нуля.  
  
### §1.6 Cover (перепеть существующий трек)  
  
`POST /generate/cover { source_audio_url | upload, style, vocals }` — берёт чужой трек, делает кавер в новом стиле/голосе.  
  
**Бизнес-применение:**  
- Юзер приносит свою рыбу-демо → получает студийную версию.  
- Перепеть классическую песню на корпоратив с упоминанием компании в тексте.  
- Создать «парный» трек: оригинал + версия от другого пола.  
- **Цена:** +199 ₽ за cover, требует прав на исходник.  
  
### §1.7 Stems separation (пост-обработка)  
  
После генерации можно получить отдельные дорожки: вокал, барабаны, бас, клавишные.  
  
**Бизнес-применение:** B2B (DJ, продакшен-студии). **+499 ₽** за stems-пакет.  
  
### §1.8 Inpainting (точечная правка) — V4.5+  
  
`POST /generate/inpaint { source_id, replace_section: [30, 50], new_prompt }` — заменяет участок трека (например, плохо спетый припев) без полной перегенерации.

### §3.2 Реализация для GPTunnel-Suno (скетч)  
  
```typescript  
// modules/music/providers/gptunnelSuno.ts  
import { env } from '../../../core/env.js';  
import { ExternalServiceError } from '../../../core/errors.js';  
  
class GPTunnelSunoProvider implements MusicProvider {  
  private base = env.GPTUNNEL_BASE_URL;  
  private key = env.GPTUNNEL_API_KEY;  
  
  async generate(input: GenerateInput): Promise<GenerateOutput> {  
    const body = this.buildGenerateBody(input);  
    return this.callWithRetry('POST', '/suno/generate', body);  
  }  
  
  async extend(input: ExtendInput) {  
    return this.callWithRetry('POST', '/suno/extend', {  
      task_id: input.sourceTaskId,  
      continue_at: input.extendAtSec,  
      prompt: input.promptHint,  
    });  
  }  
  
  async cover(input: CoverInput) {  
    return this.callWithRetry('POST', '/suno/cover', {  
      audio_url: input.sourceUrl,  
      style: this.serializeStyles(input.styles),  
      preserve_lyrics: input.preserveLyrics ?? true,  
    });  
  }  
  
  private buildGenerateBody(input: GenerateInput) {  
    return {  
      prompt: input.prompt,  
      lyrics: this.injectStructuralTags(input.lyrics, input),  
      style: this.serializeStyles(input.styles, input.vocals),  
      negative_prompt: input.negativePrompt,  
      persona_id: input.personaId,  
      make_instrumental: input.instrumental ?? false,  
      model: input.model ?? 'v4.5',  
    };  
  }  
  
  private serializeStyles(styles?: Array<{name: string; weight?: number}>, vocals?: any) {  
    if (!styles?.length) return undefined;  
    const parts = styles.map(s =>  
      s.weight && s.weight !== 1 ? `${s.name}:${s.weight}` : s.name  
    );  
    if (vocals?.gender) {  
      const w = vocals.weight ?? 1;  
      parts.push(`${vocals.gender} vocals${w !== 1 ? ':' + w : ''}`);  
    }  
    return parts.join(', ');  
  }  
  
  // Авто-добавление tempo/mood/key в lyrics, если переданы отдельно  
  private injectStructuralTags(lyrics: string | undefined, input: GenerateInput): string | undefined {  
    if (!lyrics) return undefined;  
    const meta: string[] = [];  
    if (input.tempo) meta.push(`[Tempo: ${input.tempo} bpm]`);  
    if (input.mood) meta.push(`[Mood: ${input.mood}]`);  
    if (input.key) meta.push(`[Key: ${input.key}]`);  
    if (meta.length === 0) return lyrics;  
    // Если в lyrics уже есть теги, не дублируем  
    if (/^\\s*\\[Tempo|^\\s*\\[Mood|^\\s*\\[Key/m.test(lyrics)) return lyrics;  
    return meta.join(' ') + '\\n\\n' + lyrics;  
  }  
  
  private async callWithRetry(method: string, path: string, body: any, retries = 3) {  
    let lastErr: any;  
    for (let i = 0; i < retries; i++) {  
      try {  
        const res = await fetch(this.base + path, {  
          method,  
          headers: {  
            'Authorization': `Bearer ${this.key}`,  
            'Content-Type': 'application/json',  
          },  
          body: JSON.stringify(body),  
        });  
        if (res.status === 429 || res.status === 503) {  
          await this.sleep(1000 * Math.pow(2, i));  
          continue;  
        }  
        if (!res.ok) throw new ExternalServiceError('Suno', `HTTP ${res.status}`);  
        return await res.json();  
      } catch (e) {  
        lastErr = e;  
        await this.sleep(500 * Math.pow(2, i));  
      }  
    }  
    throw new ExternalServiceError('Suno', String(lastErr));  
  }  
  
  private sleep(ms: number) { return new Promise(r => setTimeout(r, ms)); }  
}  
```

## §6. ОЧЕРЕДЬ И ПРИОРИТЕТЫ  
  
### §6.1 Базовая очередь  
  
`generations.priorityUntil` уже существует. Логика:  
- Платные треки имеют `priorityUntil = now + 5 min` → обрабатываются первыми.  
- Бесплатные / демо — обычная FIFO очередь.  
  
### §6.2 Rate limit Suno  
  
GPTunnel-Suno имеет свой rate limit (например, 10 параллельных генераций). Локальная очередь:  
  
```typescript  
// modules/music/queue.ts  
const MAX_CONCURRENT_SUNO = parseInt(env.SUNO_MAX_CONCURRENT || '5');  
const semaphore = new Semaphore(MAX_CONCURRENT_SUNO);  
  
async function processGeneration(genId: number) {  
  await semaphore.acquire();  
  try {  
    // вызов Suno  
  } finally {  
    semaphore.release();  
  }  
}  
```  
  
### §6.3 Watcher (опрос статуса)  
  
Сейчас опрос идёт интервалами. v304 — long-polling с adaptive backoff:  
- Первые 30 сек — каждые 5 сек.  
- 30-120 сек — каждые 10 сек.  
- 120-300 сек — каждые 20 сек.  
- > 300 сек — раз в минуту.  
- > 10 мин — таймаут, refund.

## §7. КАЧЕСТВО И ФИЛЬТРАЦИЯ  
  
### §7.1 Авто-проверка результата  
  
Прежде чем отдать пользователю:  
1. Длительность ≥ 60 сек (если короче — flag, возможно проблема).  
2. Уровень громкости (RMS) в норме (-23..-14 LUFS).  
3. Отсутствие тишины > 3 сек подряд.  
4. Соответствие prompt (опционально — через CLAP-эмбеддинг + similarity).  
  
Failed checks → отправить юзеру с пометкой «возможно, потребуется регенерация» + предложить regenerate бесплатно (1 раз).  
  
### §7.2 Модерация контента  
  
Юзер может ввести в lyrics или prompt экстремистский/нелегальный текст.  
  
Pre-generation модерация:  
- Список стоп-фраз (политика, призывы к насилию, hate speech).  
- LLM-классификатор (через GPTunnel) с порогом.  
- Блокировка с понятным сообщением: «Текст не прошёл модерацию. Что нельзя писать: ...».  
  
Post-generation модерация — слушать готовые треки на наличие нецензурной лексики (Whisper STT + сравнение). Опциональная фича.
