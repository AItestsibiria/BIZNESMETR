## ФИЛОСОФИЯ ПАКЕТА  
  
| Принцип | Что это значит |  
|---|---|  
| **Тонкое ядро + плагины** | Всё новое (агенты, чат-бот, аналитика, интеграции) — плагины через единый Module API. Ядро не трогаем. |  
| **Event-driven** | Модули общаются через events (`user.registered`, `gen.completed`, `payment.succeeded`). Подписка декларативна. |  
| **Конфигурируемость без релиза** | Цены, промокоды, шаблоны писем, скрипты бота — в БД, меняются из админки. |  
| **Observability по умолчанию** | Каждое действие → событие → метрика. Нет «слепых зон». |  
| **One-command deploy** | Любой деплой — одна команда из Drive. Никаких ручных шагов. |  
| **Без vendor lock** | Suno за абстракцией — можно поменять на Mubert/Riffusion за 1 файл. Robokassa за абстракцией — можно YooKassa/CryptoCloud. |

## ИЗМЕНЕНИЯ В БД (краткий список, полная версия — в 07)  
  
**Новые таблицы:**  
1. `events` — единая шина событий (для агентов и аналитики)  
2. `leads` — посетители ДО регистрации  
3. `agent_actions` — кто из агентов что сделал, для аудита  
4. `support_tickets` — обращения в поддержку  
5. `ticket_messages` — переписка по тикету  
6. `chatbot_sessions` — сессии чат-бота  
7. `chatbot_messages` — все сообщения бота и юзера  
8. `invoices` — счета на оплату для юрлиц  
9. `documents` — генерируемые PDF (договор, оферта, акт)  
10. `email_outbox` — очередь исходящих писем  
11. `email_inbox` — распарсенные входящие  
12. `notifications` — единая шина оповещений (push/sms/tg/email)  
13. `pixel_events` — события для пикселей (для server-side tracking)  
14. `tracking_attribution` — маркетинговая атрибуция (UTM, referrer, fbclid и т.д.)  
15. `audience_segments` — сегменты аудитории для retargeting  
16. `personas` — Suno persona ID (привязка к юзеру)  
17. `gen_extensions` — extend/cover-связки треков  
18. `feature_flags` — флаги фич, A/B тесты  
19. `plugins_registry` — какие плагины подключены, их версии  
20. `gen_stats_daily` — агрегаты для ленты (вместо тяжёлых JOIN)  
  
**Расширения существующих:**  
- `users`: `+lead_id`, `+lifecycle_stage`, `+lead_score`, `+rfm_recency`, `+rfm_frequency`, `+rfm_monetary`, `+ban_reason`, `+marketing_opt_in`, `+telegram_chat_id`, `+vk_user_id`, `+email_verified_at`  
- `generations`: `+persona_id`, `+structural_tags` (JSON), `+vocal_weights` (JSON), `+source_gen_id` (для extend/cover), `+suno_model` (v3.5/v4/v4.5)  
- `payments`: `+invoice_id`, `+receipt_url`, `+attribution_id`
