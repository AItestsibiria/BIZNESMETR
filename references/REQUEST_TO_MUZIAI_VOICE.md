## Контекст в одну строку

Делаем единую точку входа в Telegram, через которую CEO общается со
своим «вторым мозгом» (задачи, календарь, почта, проекты — включая

уже работает; нужен голосовой.

У вас этот контур доведён до продакшена — просим описать решение и
поделиться доступом к ключам.

## Часть А · Голосовой контур

Скажите, что у вас используется. Каждый пункт = одна-две строки ответа,
не нужны пояснения «почему», только «как».

### 1. STT (распознавание)

- **Провайдер:** OpenAI Whisper API / whisper.cpp self-host / Yandex
  SpeechKit / Salute / другое — что?
- **Модель / endpoint:** какая версия / какой URL.
- **Параметры:** `language`, `prompt`, `temperature`, `response_format`,
  таймауты — что выставляете отлично от дефолтов.
- **Перекодировка:** Telegram отдаёт `voice` как `.oga` (opus). Едите
  как есть или прогоняете через ffmpeg? Если ffmpeg — какие флаги.
- **Длинные / большие записи:** что делаете с >10 мин или >25 МБ?
  Резка на части / VAD / отказ?
- **Лайфхаки под русский:** prompt-bias, словари терминов,
  post-processing — что заметно улучшило точность?

### 2. TTS (голосовой ответ)

- **Провайдер:** OpenAI TTS / ElevenLabs / Yandex / Salute / другое?
- **Голос/voice id:** какой используете для Тани (если у вас женский
  ассистент-голос — тот же возьмём).
- **Скорость / тембр / тон:** настройки.
- **Стриминг:** генерите целиком и отправляете в Telegram одним файлом,
  или нарезаете на чанки?
- **Цена/латентность:** ориентировочные секунды + рубли на одно
  голосовое ответное сообщение.

### 3. Поведение в Telegram

- Как у вас оформляются голосовые ответы — `sendVoice` (opus в `.oga`)
  или `sendAudio`?
- Если присылают не-voice (музыка, видеосообщение, переслали аудио из
  Apple Music) — что делаете: транскрибируете / игнорите / отвечаете
  «не понимаю»?

### 4. Файлы, которые помогут больше всего

Если есть возможность — пришлите:

- `voiceHandler.ts` / `audio_handler.py` (или как у вас зовётся —
  модуль, который ловит `voice`/`audio` события и делает STT)
- модуль интеграции с STT-провайдером (клиент)
- модуль интеграции с TTS, если есть
- любые ffmpeg-снипеты, если используете

Файлы в любом виде: архив, gist, paste в чат с префиксом
`# path: src/...`.

## Часть Б · Общие ключи API

Сейчас у нас в `.env.example` есть слоты под:

| Переменная | Что |
|---|---|
| `OPENAI_API_KEY` | Whisper (STT), потенциально TTS и embeddings |
| `ANTHROPIC_API_KEY` | Claude — основной мозг ассистента |
| `GOOGLE_APPLICATION_CREDENTIALS` | Sheets, Calendar, Gmail (отдельный SA) |
| `GITHUB_TOKEN` | PAT для тулз `github_my_prs` / `github_my_issues` |
| `TELEGRAM_BOT_TOKEN` | Свой отдельный бот Novo AI |

Просьба: **подсветите**, какие из этих ключей у MuziAI уже выпущены и
можно ли их **переиспользовать** для Novo AI (вписать в `.env` нашего
сервера), а какие нужно выписать отдельно.

Для каждого общего ключа желательно:

- **Откуда взять** — кто администратор / в каком password-manager'е
  лежит / куда обратиться;
- **Лимиты** — не упрёмся ли мы в rate-limit, делая параллельный
  трафик с вашим продакшеном;
- **Изоляция** — нужны ли отдельные **org / project** внутри одного
  аккаунта (Anthropic / OpenAI поддерживают), чтобы биллинг и логи
  Novo AI не смешивались с MuziAI.

### Безопасность передачи

- **Не присылайте ключи в чат / Telegram / открытую почту.**
- Каналы: 1Password / Bitwarden shared item; private gist на одно
  чтение; PGP-зашифрованное письмо; live-сессия с экраном.
- Любой ключ, переданный в открытом виде, мы будем считать
  скомпрометированным и просить выпустить новый.

## Что мы возвращаем взамен

Когда голосовой контур и общие ключи будут — мы у себя:

- внедрим ваш STT/TTS в `src/integrations/voice.ts` (точки замены уже
  помечены `// TODO(muziai/voice)`);
- если у вас другой Telegram-паттерн обработки voice — заменим стуб в
  `src/messengers/telegram.ts` (отмечен `// TODO(muziai)`);
- отдадим обратно унифицированный `MessengerAdapter` (Telegram + MAX
  через один интерфейс), если у вас тоже планируется MAX;
- покажем дашборд с метриками потребления токенов (вдруг пригодится
  для контроля общего биллинга по shared-ключам).
