\# 08. АУДИТ ПЛЕЕР + БИБЛИОТЕКА + ПЛЕЙЛИСТЫ  
\#\# С ПОЗИЦИИ ЗАЯДЛОГО SUNO-КРЕАТОРА  
  
\*\*Дата:\*\* 2026-05-04  
\*\*Контекст:\*\* Профессионал, который генерирует 30–100 треков в неделю, ведёт несколько проектов (свадьбы, корпоративы, личное), сравнивает версии, экспортирует в DAW (Ableton/Logic), шарит черновики клиентам.  
\*\*Метод:\*\* функциональный аудит против стандартов Suno, Apple Music, Spotify, SoundCloud + специфика AI-music workflow.  
  
\---  
  
\#\# §0. РЕЗЮМЕ — 7 КРИТИЧЕСКИХ ДЫР  
  
Без этих фичей профи-креатор Suno уйдёт обратно в suno.com/aibasement через неделю.  
  
1\. \*\*Хранится только один из двух clip'ов от Suno.\*\* Suno на каждый запрос отдаёт \*\*2 варианта\*\* (clip A + clip B). Сейчас в \`result\_data.result\[0\]\` берётся первый, второй теряется. Это потеря 50% выходного контента и невозможность сравнить.  
2\. \*\*Нет A/B-сравнения.\*\* Невозможно мгновенно переключаться между двумя версиями (даже если бы хранились) — нужен toggle с сохранением точки воспроизведения.  
3\. \*\*Нет synced lyrics.\*\* Profi работает с текстом параллельно с прослушиванием — нужна построчная подсветка по таймкодам и редактор inline.  
4\. \*\*Нет экспорта в WAV/FLAC.\*\* Только MP3. Для сведения в DAW это блокер — потеря качества при дальнейшей обработке.  
5\. \*\*Нет очереди (queue) и continue-listening.\*\* Запустил трек, переключился на форму генерации — потерял позицию. Стандарт 2010-х.  
6\. \*\*Нет версионности и связей\*\* между extend/cover/inpaint в UI. В schema поле \`source\_gen\_id\` запланировано (файл 02), но плеер не показывает «дерево» из 1→3 версий одного трека.  
7\. \*\*Нет тегов, папок, поиска.\*\* Через 50 треков библиотека превращается в кашу. Только сортировка по дате — это кладбище, не библиотека.

\#\# §6. ПОДТВЕРЖДЁННЫЕ БАГИ ИЗ ТЕКУЩЕГО v303 (повтор из файла 01)  
  
Это не новые находки, а напоминание — что в плеерной зоне УЖЕ найдено в аудите ядра и должно быть закрыто в Спринте 1:  
  
1\. \*\*№3\*\* \`streaming.routes.ts: extractAudioUrl\` падает на старых записях, где аудио только в \`result\_data.result\[0\].audio\_url\` → фикс + миграция backfill \`result\_url\`.  
2\. \*\*№6\*\* \`MediaSession.seekto\` зависает на iOS Safari, когда \`setPositionState\` вызывается до \`loadedmetadata\` → очередь обновлений с flush на \`loadedmetadata\`.  
3\. \*\*№8\*\* \`gen\_activity.play\` пишется каждые 5 секунд, 30+ записей за трек → дедуп на \`(gen\_id, session\_id)\` с окном 30 минут + добавить \`play\_completed\_30s\` и \`play\_completed\_full\` для воронки.
